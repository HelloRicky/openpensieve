## Code of Conduct

Be respectful, inclusive, and constructive.

## How to Contribute

1. **Fork the repository**
2. **Create a feature branch** (`git checkout -b feature/amazing-feature`)
3. **Make your changes**
4. **Test thoroughly**
5. **Commit** (`git commit -m 'Add amazing feature'`)
6. **Push** (`git push origin feature/amazing-feature`)
7. **Open a Pull Request**

## Before You Submit

### Security Checklist

- [ ] No API keys, credentials, or personal data in code
- [ ] All secrets use environment variables
- [ ] `.env` is in `.gitignore`
- [ ] Test data is anonymized

### Code Quality

- [ ] Code follows existing style
- [ ] Types are properly defined (TypeScript)
- [ ] No console.log statements (use proper logging)
- [ ] Comments explain _why_, not _what_

### Testing

- [ ] Changes are tested locally
- [ ] No breaking changes (or clearly documented)

## Development Setup

See [README.md](README.md#development) for setup instructions.

## Reporting Issues

When reporting bugs, include:

- Steps to reproduce
- Expected vs actual behavior
- Environment (OS, Node version, etc.)
- Relevant logs (with sensitive data redacted)

## Feature Requests

Open an issue with:

- Clear use case
- Why it benefits users
- Proposed implementation (optional)

## Questions?

Open a discussion or reach out in the community Discord.
